Fixed in this session

For context — what we already addressed before this audit.

Fixed Chips degraded to plain text when a card wasn't focused

Non-editing cards rendered the visible string as plain text, so "GitHub ▾" / "[image]" replaced the styled chip. Now non-editing cards re-render styled chips (icon + brand color) from the serialized tokens.

Fixed Pressing Enter in the connector popup dropped you out of the card

The inline search field stealing first responder fired endEditing, tearing down the whole overlay. Editing now persists while our own search field / mention menu holds focus.

Fixed Card dragging jittered & smeared ghost text

Drag was tracked in the card's own moving coordinate space, feeding the move back into itself. Now tracked in stable window space.

Fixed Auto-grow forced a text layout pass every frame

Height re-measurement ran on every render; now gated to actual width reflows, so pan/drag pay nothing for it.

Fixed Canvas stopped panning when the cursor was over a card

Scroll/pinch are now forwarded from cards to the board so two-finger pan & zoom work everywhere.

Fixed Image tool removed · main panel enlarged

Removed the canvas image-object tool; bumped the panel size and reduced optical lift so the taller panel still fits a laptop.

A · Canvas input & gestures

High Toolbar zoom always re-centers the board ComposerCanvas.swift · zoom buttons

The +/− zoom buttons anchor at the viewport center, so the board lurches sideways when you zoom toward a specific card. (Pinch-zoom anchors at the cursor and feels right.)

Fix: anchor toolbar zoom on the current selection's center (or last cursor point) instead of viewport center.

Med No dead-zone before a click becomes a move BoardCardView.swift · CardPointerCatcher

The move catcher reacts to any sub-pixel drag, so a card can nudge on what the user meant as a click; there's no minimum-distance threshold like the marquee has (~4px).

Fix: require ~4px of movement before entering move mode, matching the marquee threshold.

Med A 1px twitch on a resize handle commits a resize + undo entry BoardCardView.swift · resizeGesture (minimumDistance: 1)

Grabbing a handle to nudge often resizes by a pixel and pushes an undo step, polluting history.

Fix: raise the resize gesture's minimum distance to 4–5px.

Med Space-to-pan has no visible state ComposerCanvas.swift · isSpacePressed

Holding space silently switches drag to pan mode; with no cursor/affordance change, an accidental space makes the select tool feel broken.

Fix: show a grab cursor or a small "hold space to pan" hint while space is down.

Med "Interactable only once selected" isn't the model yet BoardCardView.swift · CardPointerCatcher.allowsHitTesting

You asked for this directly: unselected cards still capture press/drag. Today a single press both selects and arms a move. A cleaner model: unselected = inert (clicks select, gestures pass to the canvas); selected = grab/resize.

Fix: have the canvas own hit-testing — first click selects only; movement drags only an already-selected card. (Larger change; worth a dedicated pass.)

Low "Fit board" can tuck content under the rails ComposerCanvas.swift · fitBoard

Fit uses a flat 40px margin and ignores the rail/toolbar gutters, so fitted cards can sit under the sidebar or toolbar.

Fix: include the gutter insets in the fit math.

B · Editing, focus & overlays

High Panel opens with nothing focused — you can't just type PanelController.swift:123 focusEditor

On open the first card is selected but not editing, so no text view exists; focusEditor finds nothing and the caret never appears. You have to double-click a card before you can type — a brutal first impression for a "brain dump" tool.

Fix: auto-enter editing on the first/last-edited card when the panel shows, then focus its editor.

Med First keystroke on a freshly placed card can be dropped BoardCardView.swift · enterEditing (0.06s focus delay)

Placing a text card focuses the editor on a delay; type instantly and the first character can go to the board, not the card.

Fix: focus synchronously when the editor mounts rather than via a timed dispatch.

Med Overlay anchors are captured once, not re-derived ComposerCanvas.swift · ActiveCardOverlays

Mention menu / connector search / lint popover anchor to a point captured when they open. Pan or zoom while one is open and it strands off to the side.

Fix: recompute the anchor from the live caret/flag rect each render, or close overlays on pan/zoom.

Med Lint popover flickers when crossing from underline to popover FreeWriteEditor.swift · handleHover / scheduleHide

The 0.35s grace + tight hit-box means moving the mouse onto the popover across a sliver of plain text can schedule a hide, so it blinks.

Fix: widen the hover hit-box and/or treat the popover's own area as keeping it alive.

Low Mention list doesn't scroll the selection into view on open MentionMenu.swift · onChange-only scrollTo

A long filtered list opens with item 0 selected but possibly scrolled off; first arrow-press appears to do nothing.

Fix: scroll to the selected index in .onAppear, not only on change.

C · Persistence & data-loss

High Editing a reloaded card shows raw @github tokens, not chips FreeWriteEditor.swift makeNSView · BoardViewModel CardInteraction init

Chips are an in-session layer; on reload only plain text persists. The new renderer restyles chips in the display state, but the moment you re-enter the card to edit, the editor loads literal @github:… text — no chip, and the next edit bakes it in as plain text.

Fix: re-chipify the plain text into an NSAttributedString on editor load (parse tokens → ChipFactory.make), so edit and display states match.

High Inline images are unrecoverable after reload MentionToken.swift composerPlainText · ComposerTextView image path

Only [image: name] is serialized — the on-disk PNG path isn't. After a reload (or app restart) the picture is gone, replaced by a placeholder, and the saved PNG is orphaned.

Fix: persist the attachment's file path in the serialized token and rebuild the attachment on load.

High Undo history is wiped on every board switch BoardViewModel.swift:191 loadFromStore

loadFromStore clears the undo/redo stacks. Glance at an older board and back, and you can no longer undo your edits.

Fix: keep undo/redo per board id, or at minimum don't reset when returning to the same board.

Med Undo baseline is the visible string, not the token form BoardViewModel.swift noteEdited

The "before" snapshot for an edit uses the visible text (chips collapsed to labels) rather than the serialized @token form, so undoing across a chip can revert to the wrong content.

Fix: seed undo from controller.plainText (the serialized form).

Med Prune-empty can delete a card you just started typing DumpStore.swift pruneCurrentIfEmpty

Empty-board pruning checks the persisted text mirror. Start a new card and navigate within the 0.4s autosave debounce and the just-typed content can be pruned.

Fix: capture editor state & flush before pruning, or also check live editor content.

D · Chrome, feedback & discoverability

High Deleting a past board has no confirmation HistoryBar.swift:105 hover ✕ → onDelete

A hover-revealed ✕ permanently deletes a board (and SwiftData has no trash). Easy to hit while dismissing a row; the work is just gone.

Fix: confirm on delete, or soft-delete with an undo toast.

Med Settings mislabels ⌘R as "Refine the whole draft" SettingsView.swift:55 · FloatingPanel.swift:102

⌘R actually compiles the whole board into a draft; Settings says it refines. The mismatch sets the wrong expectation.

Fix: relabel to "Compile to draft ⌘R" (matching the toolbar).

Med Copy says "Resolving connectors…" but never confirms completion ComposerCanvas.swift copyBoard

The progress toast vanishes after a fixed delay with no success/fail state, so you don't know if the resolved text reached the clipboard.

Fix: emit a terminal toast ("Copied · N connectors resolved" / error).

Med Toasts anchor to the viewport bottom and can be missed ComposerCanvas.swift toastView

Feedback sits at the bottom-center of the board area; with the bigger panel and scrolled content it can land out of easy sightline.

Fix: pin toasts to a fixed, always-visible spot (e.g. above the toolbar).

Low Shortcuts & features live only in hover tooltips Sidebar.swift · CanvasToolbar.swift

Pan-with-space, ⌘[/], group/lock, refine intents — all undiscoverable without hovering the right control.

Fix: a "?" shortcuts cheat-sheet popover, or a one-time onboarding hint.

Low "New dump" naming is opaque HistoryBar.swift

"Dump" leaks internal vocabulary; users think in "boards" / "drafts".

Fix: rename to "New board" (or "Start writing").

Recommended order

  1. Focus on open — auto-edit the first card so you can type immediately. Highest impact-per-effort; it's the first thing every session.
  2. Re-chipify on load + persist image paths — make tags/images survive reload in both edit and display. Core promise of the connector feature.
  3. Confirm board deletion — cheap, prevents irreversible data loss.
  4. The "select-then-interact" canvas model — unselected cards inert, canvas owns hit-testing. Removes a whole class of gesture-conflict bugs.
  5. Per-board undo + cursor-aware snapshots — stop losing undo on navigation.
  6. Toolbar zoom anchoring, drag/resize dead-zones, label/feedback fixes — polish pass.
